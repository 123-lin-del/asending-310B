## 一、项目概述

### 1.1 项目背景

随着边缘智能计算的发展，在嵌入式设备上实现实时目标检测与跟踪已成为计算机视觉领域的重要应用方向。本项目基于昇腾310B（OrangePi AIpro）开发板，构建一套完整的"检测—跟踪—云台随动"闭环系统。

### 1.2 系统目标

系统的核心工作流程如下：

1. **摄像头捕捉**：USB摄像头实时采集视频画面
2. **目标检测**：MobileNet-SSD 轻量级模型在 NPU 上进行推理，检测画面中的人物并输出边界框坐标
3. **目标跟踪**：DeepSORT 跟踪器（卡尔曼滤波 + 匈牙利算法）对检测目标分配稳定 ID，在连续帧中维持轨迹
4. **坐标提取**：从跟踪目标的边界框中提取中心点坐标 `(cx, cy)`
5. **串口传输**：将中心坐标通过 UART 串口发送至 STM32F103C8T6
6. **云台控制**：STM32 根据坐标偏差计算 PID 控制量，驱动 SG90 舵机云台转动，使摄像头始终跟随目标运动

### 1.3 开发性质

本项目为**二次开发**——CANN 8.0 开发环境已搭建完成，基础检测与跟踪框架代码由老师提供并烧录至开发板。我们的工作重心在于：理解现有代码、调优参数、完成 STM32 云台控制开发，以及全链路联调。

### 1.4 开发策略

采用**分阶段推进**策略：

> **第一阶段（优先）**：集中精力跑通并调稳检测+跟踪基础功能，确保跟踪 ID 稳定、轨迹连续，达到中等至良好的考核标准。
> **第二阶段（冲刺）**：在基础功能稳固后，推进 STM32 云台控制，实现摄像头随动跟踪，冲击优秀评级。

### 4.2 数据流详解

整个系统的数据流可以概括为以下六个环节：

**环节1：图像采集**
USB 摄像头以 640×480（或更高分辨率）采集视频帧，通过 OpenCV 读入系统。

**环节2：目标检测（MobileNet-SSD）**
每一帧图像经过预处理（resize 到 320×320、BGR→RGB、归一化、标准化、HWC→NCHW）后送入 MobileNet-SSD 模型，在昇腾 NPU 上完成前向推理。模型输出一组候选框，经解码和 NMS 后处理得到最终的检测结果（边界框坐标 + 类别 + 置信度）。

**环节3：目标跟踪（DeepSORT）**
检测结果传入 DeepSORT 跟踪器。跟踪器对每条已有轨迹执行卡尔曼滤波预测，然后通过 IOU 矩阵 + 类别兼容矩阵 + 匈牙利算法进行检测与轨迹的全局最优匹配，最后更新轨迹状态、创建新轨迹、删除失效轨迹。

**环节4：坐标提取与串口发送**
从当前帧中 ID 稳定的跟踪目标中提取边界框中心坐标 `(cx, cy)`。昇腾端通过 Python `pyserial` 库将坐标按约定帧格式经 UART 发送至 STM32：

```
帧格式：[帧头 0xAA] [cx 高字节] [cx 低字节] [cy 高字节] [cy 低字节] [校验和] [帧尾 0x55]
波特率：115200 bps
```

**环节5：STM32 接收与 PID 计算**
STM32 通过 USART 中断接收串口数据，解析出目标中心坐标。以画面中心 `(320, 240)` 为期望位置，计算目标位置与画面中心的偏差，通过 PID 控制算法输出水平轴和俯仰轴的控制量：

```
error_x = cx - 320     （水平偏差）
error_y = cy - 240     （俯仰偏差）

pwm_h += Kp * error_x + Ki * ∑error_x + Kd * Δerror_x
pwm_v += Kp * error_y + Ki * ∑error_y + Kd * Δerror_y
```

**环节6：云台驱动**
STM32 的 TIM 定时器产生两路 50Hz PWM 信号（脉宽 0.5ms~2.5ms 对应 0°~180°），分别驱动水平轴和俯仰轴的 SG90 舵机，使摄像头跟随目标运动。

### 4.4 二次开发工作内容

#### 第一阶段（优先）：检测 + 跟踪基础功能

> 目标：跑通并调稳检测+跟踪链路，确保单帧检测准确、连续帧跟踪 ID 稳定。

**步骤1：三人共同理解代码（6/30）**

三人一起阅读老师提供的代码，理清以下两条主链路：

- **检测链路**（`detection_app.py`）：图像预处理 → 加载 OM 模型 → NPU 推理 → 结果解码 → NMS 后处理 → 绘制检测框
- **跟踪链路**（`tracking_app.py`）：检测结果输入 → 卡尔曼滤波预测 → IOU 矩阵计算 → 匈牙利算法匹配 → 轨迹创建/更新/删除 → 绘制跟踪框与 ID

每人各自跑一遍程序，记录当前检测效果和跟踪表现。

**步骤2：检测调优（章涵，7/1）**

- 调整 `--score-threshold`（默认 0.5，尝试 0.3~0.7 区间）
- 调整 `--nms-threshold`（默认 0.5，尝试 0.4~0.6）
- 测试不同光照条件、不同距离下的检测效果
- 选定最优参数组合并记录

**步骤3：跟踪调优（陈一帆，7/1~7/2）**

- 调整 `--track-max-age`（轨迹最大丢失帧数，默认 30，尝试 60/120）
- 调整 `--track-min-hits`（轨迹最少命中次数，默认 3，尝试 1/5）
- 调整 `--track-iou-threshold`（IOU 关联阈值，默认 0.3，尝试 0.4/0.5）
- 调整 `--track-center-distance-threshold`（中心距离补充匹配阈值）
- 使用 `--track-classes person` 只跟踪行人，降低其他类别干扰
- 记录最优参数组合

**步骤4：基础验证（全体，7/2 上午）**

- 摄像头实时场景下连续运行 5 分钟以上
- 统计 ID 切换次数、跟踪丢失情况
- 记录当前 FPS 作为性能基线
- 确保达到「可演示」状态

#### 第二阶段（冲刺）：STM32 云台控制 + 全链路联调

> 目标：在基础跟踪稳定的前提下，增加云台随动功能，冲击优秀评级。

**步骤5：STM32 云台控制开发（林棉峰，7/2~7/4）**

1. **硬件连接**
   - SG90 舵机信号线 → STM32 TIM PWM 输出引脚（两路：水平 PA0 + 俯仰 PA1）
   - ST-Link → STM32 SWD 调试接口
   - 昇腾310B UART TX → STM32 USART1 RX（通过 USB 转 TTL 模块）
2. **PWM 驱动**：配置 TIM 产生 50Hz PWM，脉宽 0.5ms~2.5ms 对应 0°~180°
3. **串口接收**：USART1 中断接收昇腾310B 发送的目标坐标
   - 帧格式：帧头(0xAA) + cx(16bit) + cy(16bit) + 校验和 + 帧尾(0x55)
   - 波特率：115200
4. **PID 控制**：以画面中心 (320, 240) 为期望位置，目标偏差为输入，输出 PWM 控制量
5. **单独测试**：先用串口调试助手模拟坐标发送，验证舵机响应方向与幅度正确

**步骤6：全链路联调（全体，7/4~7/6）**

1. **协议对齐**：确认昇腾端串口发送格式与 STM32 接收解析完全一致
2. **闭环测试**：摄像头 → 检测 → 跟踪 → 坐标提取 → 串口 → STM32 → 舵机云台，完整链路验证
3. **延迟优化**：调整推理帧率、串口波特率、PID 参数整定，减少跟踪滞后
4. **边界测试**：目标快速移动、目标短暂离开画面后重新出现、多目标交叉场景

## 八、核心算法原理简述

### 8.1 MobileNet-SSD 检测原理

MobileNet 利用深度可分离卷积（Depthwise + Pointwise）大幅降低参数量和计算量，非常适合边缘设备。SSD 作为一阶段检测器，在多个尺度特征图上通过预定义的先验框直接回归目标位置和类别，无需候选区域生成，推理速度快，适合实时场景。

两者结合后，MobileNet-SSD 能够以较低的算力完成实时目标检测，为本项目的跟踪环节提供稳定、及时的帧级观测输入。

### 8.2 DeepSORT 跟踪原理

DeepSORT 在 SORT 基础上引入外观特征建模，本案例采用简化版本，保留四个核心机制：

- **卡尔曼滤波**：对每条轨迹维护状态估计（位置、速度），通过预测-更新两步平滑观测噪声，提供帧间运动预测
- **匈牙利算法**：基于 IOU 矩阵和类别兼容矩阵，实现检测与轨迹的全局最优匹配，避免贪心匹配导致的 ID 交换
- **IOU 匹配**：以交并比作为几何相似度的主要度量
- **轨迹生命周期管理**：通过 max_age（最大丢失帧数）和 min_hits（最少命中次数）控制轨迹的创建与删除

### 8.3 PID 控制原理

PID 控制器根据目标位置与画面中心的偏差，计算云台舵机的控制量：

- **P（比例）**：直接与当前偏差成正比，提供即时响应
- **I（积分）**：累积历史偏差，消除稳态误差
- **D（微分）**：反映偏差变化趋势，抑制超调和振荡

三个分量加权求和后输出为 PWM 脉宽调整量，驱动水平和俯仰两路舵机。
